TODO: Fitur Pembuka & Penutup (Playlist Otomatis sebelum/sesudah jadwal)

Branch: dev (existing)
Status: ⬜ Belum dimulai

Deskripsi:
- Putar rangkaian audio aset (playlist) secara otomatis **sebelum bell pertama**
  pada hari sekolah aktif (pembuka).
- Putar rangkaian audio aset (playlist) secara otomatis **setelah bell terakhir**
  (penutup).
- Setelah semua aset penutup selesai diputar, jalankan **action** tertentu
  (misal: shutdown komputer client, restart, atau command kustom).
- Pengaturan dilakukan di panel admin dengan tabel terpisah.

Phase 1: Database — Tabel bell_playlists
-------------------------------------------------------------------------------

⬜ 1.1 Buat migration: create_bell_playlists_table
       Kolom:
       - id (bigIncrements)
       - type (enum: 'opening', 'closing')
       - name (string) — label/nama playlist
       - audio_assets (json) — daftar file audio dalam urutan
         [{asset_id, filename, name, duration?}, ...]
       - time_range_start (time, nullable) — jam mulai berlaku
         (misal: pembuka mulai dari jam 06:30 setiap hari)
       - time_range_end (time, nullable) — jam selesai berlaku
         (misal: penutup hanya berlaku sampai jam 18:00)
       - is_active (boolean, default true)
       - day_of_week (json, nullable) — filter hari tertentu
         (null = semua hari aktif)
       - action_after (string, nullable) — command/shell setelah

       - action_after_delay (integer, default 0) — delay dalam detik
         sebelum action dijalankan
       - order (integer) — urutan jika multiple playlist
       - created_at, updated_at

Phase 3: Backend — Scheduler (ProcessBellSchedules)
-------------------------------------------------------------------------------

⬜ 3.1 Setiap menit, scheduler cek:
       - Apakah sudah waktunya **pembuka**?
         (time_range_start ≤ now ≤ time_range_start + offset_playlist)
       - Apakah sudah waktunya **penutup**?
         (time_range_start ≤ now ≤ time_range_start + offset_playlist)

⬜ 3.2 Logic pembuka:
       - Cari playlist type='opening' yang aktif dan hari ini cocok
       - Cek apakah sudah pernah diputar hari ini (cache key)
       - Jika belum, broadcast PlaylistStarted untuk item pertama
       - Setiap menit, broadcast item berikutnya
       - Setelah semua item, tandai selesai

⬜ 3.3 Logic penutup:
       - Sama seperti pembuka, tapi setelah semua item selesai
       - Broadcast PlaylistFinished dengan action_after
       - Jika action_after_delay > 0, tunggu
       - Broadcast ShutdownRequested

⬜ 3.4 Grace period:
       - Jika playlist terlewat karena baru dibuat, grace period
         sama seperti schedule biasa (2 menit)

Phase 6: Action Execution — Endpoint Shutdown (Client-side Agent)
-------------------------------------------------------------------------------

⬜ 6.1 Untuk action shutdown/restart di client PC:
       - Diperlukan agent/client-side yang bisa menjalankan perintah
         sistem. Opsi:
         a) Browser tidak bisa shutdown langsung (sandboxed).
            Alternatif: fetch ke endpoint di server → server SSH
            ke client atau jalankan script lokal.
         b) Buat service kecil (Python/Bash) di client yang polling
            endpoint di server atau listen WebSocket juga.
         c) Fallback: browser close window saja (+ instruksi manual
            untuk shutdown).

⬜ 6.2 Implementasi paling sederhana:
       - Server menyediakan endpoint GET /api/shutdown
       - Client agent (script bash/python) polling endpoint ini
         setiap N detik
       - Jika ada perintah, jalankan (shutdown now, reboot, dll)
       - Atau: manfaatkan WebSocket yang sudah ada (ShutdownRequested
         event → kirim ke agent)

⬜ 6.3 Agent script (opsional):
       - Python script kecil yang WebSocket client ke Reverb
         (pakai websockets library)
       - Atau bash script polling HTTP endpoint
